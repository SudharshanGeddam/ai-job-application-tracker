---
title: "Generate Cover Letter"
api: "POST /api/applications/{id}/cover-letter"
description: "Generate a personalized cover letter using a two-stage AI pipeline: talking points extraction → full draft."
authMethod: "bearer"
---

<Note>
  **JD Match Analysis must be completed first.** Run `POST /:id/analyze-jd` before calling this endpoint. The cover letter generation uses the match analysis as context.
</Note>

## Path Parameters

<ParamField path="id" type="string" required>
  The UUID of the application.
</ParamField>

This endpoint requires no request body — it uses data already stored on the application:
- `resumeText` — the extracted resume text
- `jdText` — the stored job description
- `jdMatchResult` — the AI match analysis

## How It Works

```
LLM Call 1: Extract 2-3 talking points grounded in resume evidence
LLM Call 2: Write full cover letter draft using those talking points
```

Both outputs are saved to the application record.

## Response

<ResponseField name="data" type="object">
  <Expandable title="Cover letter result">
    <ResponseField name="talkingPoints" type="object">
      Structured talking points used to ground the draft.
    </ResponseField>
    <ResponseField name="draft" type="object">
      <Expandable title="Draft fields">
        <ResponseField name="greeting" type="string">Opening greeting.</ResponseField>
        <ResponseField name="opening" type="string">Opening paragraph.</ResponseField>
        <ResponseField name="body" type="array">Array of body paragraph strings.</ResponseField>
        <ResponseField name="closing" type="string">Closing paragraph.</ResponseField>
        <ResponseField name="signature" type="string">Sign-off line.</ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash curl
curl -X POST http://localhost:3000/api/applications/uuid-here/cover-letter \
  -H "Authorization: Bearer <token>"
```
</RequestExample>
